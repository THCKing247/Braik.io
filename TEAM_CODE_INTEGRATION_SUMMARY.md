# Team Code Integration Summary

## Overview

This document outlines the implementation of the head coach's team code system, allowing all user roles (except head coaches) to connect to teams using the team code generated by the head coach.

## Changes Made

### 1. Team Join Route (`app/api/team/join/route.ts`)

**Updated to support head coach's team code (`team_id_code`):**

- **Priority 1:** Player-specific invite codes (`players.invite_code`)
  - Links user to existing coach-created player record
  - Updates player record with `user_id` when claimed

- **Priority 2:** Head coach's team code (`teams.team_id_code`) ⭐ **NEW**
  - Main team code that head coaches generate and share
  - Works for all roles: players, parents, assistant coaches
  - No expiration or usage limits (unlike legacy invite codes)

- **Priority 3:** Legacy team invite codes (`invites.code`)
  - Fallback for existing invite system
  - Supports expiration and usage limits

**Additional Updates:**
- ✅ Prevents head coaches from joining other teams (they create their own)
- ✅ Validates code format (normalized to uppercase)
- ✅ Updates `profiles.team_id` to link user to team
- ✅ Preserves user's role when joining (player, parent, assistant_coach)

### 2. Role-Specific Portal Access

**Already implemented in previous changes:**
- ✅ All roles can login without team code
- ✅ Dashboard shows welcome screen for users without teams
- ✅ Users can connect to team via team code from dashboard
- ✅ Role-specific portals work after team connection:
  - **Players:** Access roster, schedule, messages
  - **Parents:** Access linked player data, messages
  - **Assistant Coaches:** Access coaching tools, roster management
  - **Head Coaches:** Create teams, manage settings

## How It Works

### For Head Coaches

1. **Generate Team Code:**
   - Head coach goes to Settings → Team Code
   - Clicks "Generate Team Code"
   - System creates unique `team_id_code` (8 characters, alphanumeric)
   - Code is displayed and can be copied/shared

2. **Share Team Code:**
   - Head coach shares the code with:
     - Assistant coaches
     - Players
     - Parents
   - Code can be shared via:
     - Email
     - Text message
     - QR code (if implemented)
     - In-person

### For Users (Players, Parents, Assistant Coaches)

1. **Login/Signup:**
   - User signs up or logs in (team code not required)
   - User sees welcome screen on dashboard

2. **Connect to Team:**
   - User enters team code on dashboard
   - System validates code against `teams.team_id_code`
   - If valid, user is linked to team via `profiles.team_id`
   - User's role is preserved (player, parent, assistant_coach)

3. **Access Portal:**
   - After connection, user sees full team dashboard
   - Access is role-specific:
     - Players see player portal
     - Parents see parent portal
     - Assistant coaches see coaching portal

## Code Validation Flow

```
User enters code
    ↓
Normalize to uppercase
    ↓
Check 1: players.invite_code (player-specific)
    ↓ (if not found)
Check 2: teams.team_id_code (head coach's team code) ⭐
    ↓ (if not found)
Check 3: invites.code (legacy invite system)
    ↓ (if not found)
Return error: "That code is not valid"
```

## Database Schema

### Teams Table

The `teams` table already has the required columns:

```sql
team_id_code text UNIQUE  -- Head coach's main team code
player_code text UNIQUE   -- Player-specific code (future use)
parent_code text UNIQUE   -- Parent-specific code (future use)
```

**Migration:** No new migration needed - columns already exist in `20260303000000_profiles_and_auth_sync.sql`

### Profiles Table

Users are linked to teams via:

```sql
profiles.team_id uuid  -- References teams.id
profiles.role text    -- User's role (player, parent, assistant_coach, head_coach)
```

## Security & Access Control

### RBAC (Role-Based Access Control)

- Uses `profiles.team_id` and `profiles.role` as source of truth
- `getUserMembership()` checks:
  1. `profiles.team_id` matches requested team
  2. `profiles.role` determines permissions
  3. `teams.created_by` for head coach detection

### Team Code Security

- **No expiration:** Team codes don't expire (unlike legacy invites)
- **No usage limits:** Team codes can be used unlimited times
- **Unique codes:** Each team has one unique `team_id_code`
- **Case-insensitive:** Codes are normalized to uppercase for matching

### Restrictions

- ✅ Head coaches cannot join other teams (they create their own)
- ✅ Users can only join one team at a time
- ✅ Users already connected to a team cannot join another

## API Endpoints

### Generate Team Code
```
POST /api/roster/generate-codes?teamId={teamId}
```
- **Auth:** Head coach only (requires "manage" permission)
- **Response:** `{ playerCode, parentCode, teamIdCode }`

### Get Team Codes
```
GET /api/roster/codes?teamId={teamId}
```
- **Auth:** Team member
- **Response:** `{ teamName, playerCode, parentCode, teamIdCode }`

### Join Team
```
POST /api/team/join
Body: { code: "ABC12345" }
```
- **Auth:** Authenticated user (not head coach)
- **Response:** `{ success: true, teamId, teamName }`

## Troubleshooting

### Issue: "That code is not valid"
**Possible causes:**
- Code was mistyped (check for typos)
- Code hasn't been generated yet (head coach needs to generate it)
- Code is from a different team

**Solution:**
- Verify code with head coach
- Ensure code is entered in uppercase (auto-normalized)
- Check that head coach has generated the code

### Issue: "Head coaches cannot join other teams"
**Cause:** Head coaches are restricted from joining teams (they create their own)

**Solution:** This is expected behavior. Head coaches should create their own team instead.

### Issue: "You are already connected to a team"
**Cause:** User already has `profiles.team_id` set

**Solution:** User must leave current team before joining another (if this feature is implemented)
